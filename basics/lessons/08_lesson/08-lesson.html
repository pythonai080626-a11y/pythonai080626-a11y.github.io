while True, break & continue

Last lesson every while loop ended by itself when its condition turned False. Today we learn a second way out: a loop that says while True — it would run forever — and a break statement that jumps out of it from inside the body. We start with the classic FizzBuzz, then meet while True + break, and finish with continue — the command that skips the rest of a round without leaving the loop.

FizzBuzz

Count 1..N — but Fizz, Buzz, FizzBuzz

The most famous beginner loop in the world. Count from 1 up to a number the user picks, and for each number:

divisible by 3 → Fizz divisible by 5 → Buzz divisible by both → FizzBuzz otherwise → the number itself

Everything hangs on the modulus operator % — the remainder of a division. i % 3 == 0 means "i divides evenly by 3", i.e. it is a multiple of 3.

First — the flowchart

Read it top-to-bottom. The loop-back arrow on the far left is the while: after printing a space it jumps up and checks i <= max_number again.

The Python code
# START max_number = int(input('max number?')) i = 1 while i <= max_number: if i % 3 != 0 and i % 5 != 0: # not a multiple of 3 or 5 print(i, end="") else: if i % 3 == 0: # multiple of 3 → Fizz print('Fizz', end="") if i % 5 == 0: # multiple of 5 → Buzz print('Buzz', end="") i += 1 # the loop counter — never forget it! print(' ', end="") # one space between items print('bye bye') # STOP
Output (max number = 15)
max number?15 1 2 Fizz 4 Buzz Fizz 7 8 Fizz Buzz 11 Fizz 13 14 FizzBuzz bye bye
Why end=""? By default print jumps to a new line after every call. end="" tells it "do not move to a new line" — so all the numbers land on one row, separated only by the space we print ourselves.
Why does "FizzBuzz" come out for 15? When i is 15 both ifs inside the else are True — so Fizz is printed and then Buzz, right next to each other. Two separate ifs (not elif) is exactly what lets both fire.
Watch the multiples of 3 and 5. The first condition asks for i % 3 != 0 and i % 5 != 0 — "not a multiple of 3 and not a multiple of 5". Only then do we print the number as-is. If it fails, at least one of Fizz / Buzz applies.

while True

A Loop That Never Ends — On Purpose

Sometimes we do not know the stop condition before the loop starts — it only becomes clear in the middle of a round. For that we write a loop whose condition is simply True. True is always true, so the loop would run forever… unless something inside it uses break to jump out.

while True: <do some work> if <time to stop>: break # leave the loop right now <more work> <the program continues here after break>
break = emergency exit. The moment Python runs break, it stops the loop instantly and continues on the first line after the loop — even if we were in the middle of the body. The condition True is never re-checked, because we never go back up.

Here is the same "count 1 to 3" written both ways — with a normal condition, and with while True + break. They print exactly the same thing:

number = 1 # classic while — the condition guards the top while number <= 3: print(number) number += 1 number = 1 # while True — we break out from the inside while True: print(number) if number >= 3: break number += 1 print('Finish')
1 2 3 1 2 3 Finish
Old friends by other names. Other languages call this shape do…while or repeat…until — a loop that always runs at least once and checks the stop condition at the bottom. Python has no do…while; while True + break is how we build it.
Every while True needs a reachable break. If no break can ever run, you have written an infinite loop on purpose but with no exit — the program hangs. Stop it with the red Stop button in PyCharm (or Ctrl + C in a terminal).

Triangle Checker — Keep Going Until a Bad One

Read the three sides a, b, c of a triangle and print its perimeter (the sum of the sides). Then ask for the next triangle, again and again. Stop only when the sides do not make a valid triangle.

When is a triangle valid?
(1) every side is positive: a > 0, b > 0, c > 0
(2) each pair of sides is longer than the third: a + b > c, a + c > b, b + c > a
equilateral ✓
5, 5, 5 — every side equal
right triangle ✓
3, 4, 5 — the classic
too flat ✗
2, 2, 10 — 2 + 2 can't reach 10

The first two are real triangles → the loop prints their perimeter. The flat one fails the rule → break.

# input triangle sides a, b, c — valid ? print perimeter : stop # this is a do-while / repeat-until built with while True while True: a = float(input('side a?')) # float, so 2.5 is allowed too b = float(input('side b?')) c = float(input('side c?')) if a <= 0 or b <= 0 or c <= 0: break # a side is not positive → stop if a + b <= c or a + c <= b or b + c <= a: break # fails the triangle rule → stop print('sum of sides:', a + b + c) print('next triangle ...') print('bye bye')
Output
side a?3 side b?4 side c?5 sum of sides: 12.0 next triangle ... side a?2 side b?2 side c?10 bye bye
Two breaks, two reasons to stop. A while True can have as many exits as you like. The second triangle (2, 2, 10) breaks the rule a + b > c — 2 + 2 = 4 is not greater than 10 — so the second break fires and we never print its perimeter.

Login Screen — Ask Until Correct

Keep asking for a username and password until the user finally types the right pair. This is the everyday face of while True: retry until success, then break out.

Sign in 👋

Enter your username and password

Wrong credentials...
admin |
••••
Log in

🔑 Type admin / 1234 to break out of the loop

# START while True: user = input('username?') pwd = input('password?') if user == "admin" and pwd == "1234": break # correct → get out of the loop else: print('Wrong credentials...') # wrong → loop asks again # STOP
Output
username?guest password?0000 Wrong credentials... username?admin password?1234
Notice the break happens on success, not on failure. While the credentials are wrong the else runs and the loop simply spins again. Only the correct pair reaches break, and that is the single door out.
Passwords compared with == like this are only for learning — real programs never store or compare passwords in plain text.

Highest Grade — Now With break

Remember diagram 23 from last lesson? Find the highest grade in a class. Last time the loop condition did two jobs at once: while i < num_stud and max_grade < 100. Today we rewrite it as a while True with two separate breaks — often easier to read than a long and condition.

# find the highest grade — stop early if someone scores 100 num_stud = int(input('number of students?')) i = 0 max_grade = 0 while True: i += 1 grade = int(input('grade?')) if grade < 0 or grade > 100: print('illegal grade') elif grade > max_grade: max_grade = grade if max_grade == 100: break # perfect grade — nothing can beat it, stop if i >= num_stud: break # asked everyone — stop print(max_grade)
Output (4 students, someone scores 100)
number of students?4 grade?80 grade?100 100
The condition moved inside. Instead of the header carrying both stop-reasons, each break now owns one: "a perfect grade appeared" and "we asked every student". Same behaviour as the and version — just spelled out step by step.
Same catch as before: i += 1 runs at the top of every round, so an illegal grade still uses up one of the students. That is the class version; a stricter one would count a student only after a valid grade.
continue

Skip a Round — continue

break leaves the loop for good. continue is gentler: it drops the rest of the current round and jumps straight back up to the loop — the loop keeps going. Perfect for "if this input is junk, ignore it and ask for the next one".

break vs continue in one line:
break → leave the loop entirely.
continue → skip to the next round of the same loop.

The task: read grades and print the class average. 999 means "no more grades" (stop). Any grade outside 0–100 is invalid — we warn and ignore it, without letting it spoil the average.

# read grades until 999, ignore invalid ones, print the average number_of_students = 0 total = 0 while True: grade = int(input('Grade?')) if grade == 999: break # stop signal — leave the loop if grade < 0 or grade > 100: print('Invalid grade, ignored') continue # skip the two lines below, ask again total += grade # only valid grades reach here number_of_students += 1 print('Students:', number_of_students) if number_of_students > 0: print('Average grade:', total / number_of_students)
Output
Grade?80 Grade?200 Invalid grade, ignored Grade?90 Grade?999 Students: 2 Average grade: 85.0
Trace table

Why the last if number_of_students > 0? If the very first thing typed is 999, no grades were counted. Dividing by 0 would crash the program, so we only compute the average when at least one grade was accepted.
The GitHub file names this variable sum. sum is a built-in Python function, so here we renamed it total — a small, safe improvement.

Movie ratings

Ask the user for a movie rating again and again. Stop only when they type -1. For every rating:

  • if the number is not between 1 and 5 → ignore it (invalid score)
  • a score of 1 or 2 → add 1 to low_rate
  • a score of 3 → add 1 to medium_rate
  • a score of 4 or 5 → add 1 to high_rate

At the end, print how many valid scores were given in total, and how many were low, medium and high.

Example
rating?5 rating?3 rating?2 rating?9 rating?4 rating?1 rating?-1 Valid ratings: 5 Low (1-2): 2 Medium (3): 1 High (4-5): 2
Why 5 valid? Six numbers were typed before -1, but 9 is outside 1–5 so it is ignored. The five valid ones split into low 2 + medium 1 + high 2 = 5.
Hint: a while True loop — break when the number is -1, and continue to skip an invalid score (outside 1–5). Start four counters at 0 before the loop: valid, low_rate, medium_rate, high_rate (or just add the three rate counters — they sum to the valid count).
