Two-Digit Number — Tens vs Ones

Read a number from the user. First check it really is a two-digit number (between 10 and 99). If it is, compare the ones digit (ahadot / אחדות) with the tens digit (asarot / עשרות).

The two tricks:
a % 10 gives the ones digit — the remainder after dividing by 10.
a // 10 gives the tens digit — the whole part of the division.
For 44: 44 % 10 = 4 and 44 // 10 = 4 → equal digits.
The Python code
# START a = int(input('enter 2 digit number')) if a >= 10 and a <= 99: ahadot = a % 10 asarot = a // 10 if ahadot == asarot: print('equal digits') else: print('NOT equal digits') else: print('number should be between 10-99') # STOP
Output (user types 44)
enter 2 digit number44 equal digits
Output (user types 5)
enter 2 digit number5 number should be between 10-99

Nesting again. The second if lives inside the first one, so it is indented by 8 spaces. Python only reaches it when the number really is between 10 and 99 — that is why we never divide a number that we did not validate.

The while Loop

Until now every line ran once. A while loop repeats a block of code as long as a condition is True. It is the code version of the loop-shaped arrow we keep drawing in flowcharts.

while <condition>: <body — runs again and again> <this line runs only after the loop finished>
How Python runs it:
1. Check the condition.
2. If it is True → run the indented body, then go back to step 1.
3. If it is False → skip the body and continue after the loop.
The condition is checked before every round — so a loop can also run zero times.
Counting from 1 to 5
# START i = 1 while i <= 5: print(i) i += 1 # same as: i = i + 1 print('done') # STOP
1 2 3 4 5 done

Infinite loop! If nothing inside the body changes the condition, the loop never stops: while i <= 5: print(i) without i += 1 prints 1 forever. Every while needs something in the body that eventually makes the condition False. In PyCharm you stop a runaway program with the red Stop button.
A loop condition can use and / or too. That is exactly what we do in diagram 23: the loop keeps going while there are students left and we have not already found a perfect grade.

Highest Grade in the Class

Input the number of students in the class, then input the grade of each student, and finally print the highest grade. A grade must be between 0 and 100; anything else is an illegal grade.

The plan:
1. Input num_stud
2. i = 0, max_grade = 0
3. While i < num_stud and max_grade < 100:
    • i += 1
    • input grade
    • if the grade is illegal (< 0 or > 100) → print illegal grade
    • otherwise, if grade > max_grade → max_grade = grade
4. Print max_grade
Correction to the class diagram: in the flowchart we drew in class the 0-100 check was missing. The diagram below is the fixed version — it includes the grade < 0 or grade > 100 decision, exactly like the Python code we wrote.
The Python code
# START num_stud = int(input('number of students?')) i = 0 max_grade = 0 while i < num_stud and max_grade < 100: i += 1 grade = int(input('grade?')) if grade < 0 or grade > 100: print('illegal grade') elif grade > max_grade: max_grade = grade print(max_grade) # STOP
Output (3 students: 80, 95, 60)
number of students?3 grade?80 grade?95 grade?60 95
Trace table (num_stud = 3, grades 80, 95, 60)

Why max_grade < 100 in the loop condition? 100 is the best grade possible. The moment someone gets a 100 there is no point asking for more grades — nothing can beat it — so the loop stops early. This is a small optimization: the loop ends when the students run out or when a perfect grade shows up, whichever comes first.
Why elif and not a second if? A grade is either illegal or a candidate for the new maximum — never both. elif says "only check this when the previous condition was False", so an illegal grade can never be stored in max_grade.
Careful: i += 1 runs before we know whether the grade is legal, so an illegal grade still uses up one of the students. That is what the class code does; a stricter version would only increase i when the grade was accepted.

Keep Asking Until a Negative Number

Ask the user for a number again and again. As long as the number is 0 or bigger, ask again. The moment a negative number is entered, the loop stops and the program says goodbye.

# START number = int(input('number?')) while number >= 0: print('Enter negative number to exit') number = int(input('number?')) print('Negative number was entered') # STOP
Output
number?1 Enter negative number to exit number?2 Enter negative number to exit number?3 Enter negative number to exit number?-4 Negative number was entered
Two inputs, on purpose. The first input happens before the loop — otherwise the condition would have nothing to test. The second one is the last line of the body, so every round brings a fresh number to check. This "input before, input again at the end" shape comes back in almost every loop that reads from the user.
number >= 0 and not number < 0 are the same condition written two ways — the flowchart shows both.

Sum Numbers Until 999

Read numbers from the user until they type 999, then print the sum of everything they typed. The 999 itself is not part of the sum — it is only the signal to stop. For the input 1, 10, 999 the answer is 11.

# START number = int(input('number?')) total = 0 while number != 999: total += number number = int(input('number?')) print(total) # STOP
Output (1, 10, 999)
number?1 number?10 number?999 11
Trace table

999 is a "sentinel" — a stop value. Because total += number happens inside the loop and the loop already ended when 999 arrives, the 999 is never added. That is exactly why the input line sits at the end of the body.
Do not call the variable sum — that is the name of a built-in Python function. Use total instead.

Multiply Without Using *

Input two positive numbers a and b and print a * b — but the * operator is not allowed. The trick: multiplication is just repeated addition. 5 * 3 means 5 + 5 + 5, so add a to a total b times.

# START a = int(input('a?')) b = int(input('b?')) total = 0 while a > 0 and b > 0: total += a b -= 1 print(total) # STOP
Output (a = 5, b = 3)
a?5 b?3 15
Trace table (a = 5, b = 3)

52True5 + 5 = 10
51True10 + 5 = 15

What makes the condition False? b -= 1. Each round burns one b, so after exactly b rounds the loop ends — and a was added exactly b times. Checking a > 0 too means a zero or negative a gives 0 straight away, without ever entering the loop.
Diagram 16

Factorial (atzeret / עצרת)

The factorial of a number means: take every whole number from 1 up to that number, and multiply them all together.
In maths it is written with an exclamation mark — 5! is read "five factorial".

See it grow:
1! = 1 = 1
2! = 1 * 2 = 2
3! = 1 * 2 * 3 = 6
4! = 1 * 2 * 3 * 4 = 24
5! = 1 * 2 * 3 * 4 * 5 = 120

Notice that each answer is just the previous answer multiplied by the new number:
5! = 4! * 5 = 24 * 5 = 120. That is exactly what the loop does — one multiplication per round.

So how do we build it in code?
We need two variables:
• result — the answer we are building, starting at 1
• i — the number we are currently multiplying by, also starting at 1

Then, while i has not reached the number yet, we push i one step up and multiply it into result. When i finally equals the number, every factor from 1 to number has been used — and result holds the factorial.

# START number = int(input('number?')) i = 1 result = 1 while i < number: i += 1 result = result * i print(result) # STOP
Output (number = 5)
number?5 120
Trace table (number = 5)
i < 5 ?i becomesresult
1 < 5 → True21 * 2 = 2
2 < 5 → True32 * 3 = 6
3 < 5 → True46 * 4 = 24
4 < 5 → True524 * 5 = 120
5 < 5 → Falseloop stops, print 120
Why result = 1 and not 0? Because we multiply into it. Anything times 0 stays 0 forever. 1 is the neutral starting point for multiplication, exactly like 0 is for addition.
For number = 1 the condition 1 < 1 is False immediately, the loop runs zero times, and the program prints the correct answer 1.
Diagram 17

Sum of the Digits

Input a number and print the sum of its digits — not the number itself, but the digits it is made of, added together.
123 → 1 + 2 + 3 = 6
456 → 4 + 5 + 6 = 15

The problem: a number is not a list of digits — Python sees 456 as one single value. So we have to pull the digits out ourselves, one at a time. Luckily we already met the two operators that can do it:

The two tools:
number % 10 — the remainder of dividing by 10. It reads the last digit.
    456 % 10 = 6
number // 10 — whole division by 10. It throws the last digit away.
    456 // 10 = 45

The idea: use them together, again and again. Each round of the loop takes the last digit, adds it to a running total, and then chops that digit off the number:
456 → take 6, left with 45
45 → take 5, left with 4
4 → take 4, left with 0

The number keeps shrinking, and once it reaches 0 there are no digits left — that is what stops the loop. The total by then is 6 + 5 + 4 = 15.

# START number = int(input('number?')) total = 0 while number > 0: digit = number % 10 # the last digit total += digit number = number // 10 # cut the last digit off print(total) # STOP
Output (number = 456)
number?456 15
Trace table (number = 456)
number beforedigit (% 10)totalnumber after (// 10)
45660 + 6 = 645
4556 + 5 = 114
4411 + 4 = 150
00 > 0 is False → loop stops, print 15
The digit-peeling pattern. % 10 reads the last digit, // 10 removes it. Together they walk through a number digit by digit, and number shrinks to 0 — which is what ends the loop. Remember this pair: the palindrome exercise uses exactly the same two lines.
Diagram 20

Palindrome Number

A palindrome reads the same forwards and backwards. Words can be palindromes too — like "level" or "racecar" — and so can numbers:
2332 reversed is 2332 → palindrome ✅
232 reversed is 232 → palindrome ✅
7 reversed is 7 → palindrome ✅
23 reversed is 32 → not a palindrome ❌

The problem: Python cannot simply "read a number backwards" — so we have to build the reversed number ourselves, digit by digit, using the very same peeling trick from diagram 17: % 10 reads the last digit, // 10 chops it off.

Reversing 2332, one digit at a time:
Start with reversed = 0.
Each round we move what we already built one place to the left (multiply by 10) and drop the new digit into the empty spot on the right:

take 2 → 0 * 10 + 2 = 2
take 3 → 2 * 10 + 3 = 23
take 3 → 23 * 10 + 3 = 233
take 2 → 233 * 10 + 2 = 2332

One catch: the loop eats the number — by the end num has shrunk all the way to 0, so there is nothing left to compare with. That is why, before the loop starts, we save a copy in original. When the loop finishes we simply ask: is original equal to the reversed number?

# START num = int(input('number?')) if num < 1: print('enter positive number') else: original = num reversed_num = 0 while num > 0: last_digit = num % 10 reversed_num = reversed_num * 10 + last_digit num = num // 10 if original == reversed_num: print('palindrom') else: print('not-palindrome') # STOP
Output (num = 2332)
number?2332 palindrom
Output (num = 23)
number?23 not-palindrome
Trace table (num = 2332)

Why keep original? The loop destroys num — it shrinks to 0. Without a copy taken before the loop there would be nothing left to compare against. Saving a variable before a loop eats it is a habit worth building.
Building the reversed number: reversed_num * 10 pushes what we already have one place to the left, and + last_digit drops the new digit into the empty spot on the right.
In Python reversed is the name of a built-in function, so the code uses reversed_num. The flowchart still says reversed — flowcharts do not care, Python does.

Hint: this is diagram 23 turned upside down. Use a counter i and a while loop that runs num_stud times, and keep the answer in a variable such as min_grade.
