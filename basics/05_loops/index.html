Loops — while 🔁

Until now every line ran once. A while loop repeats a block of code as long as a condition is True — it is the Python version of the arrow that goes back up in a flowchart. Learn the loop counter, the accumulator, and how to avoid the dreaded infinite loop.

count.py
i = 1 while i <= 5: print(i) i += 1 print('done')
1

The Shape of a while Loop

A while looks exactly like an if — a condition, a colon, and an indented block. The only difference: when the block finishes, Python goes back up and checks the condition again.

while <condition>: <body — runs again and again> <this line runs only after the loop finished>
🔄 How Python runs it: ① check the condition → ② if True, run the body and go back to ① → ③ if False, skip the body and continue below the loop.
⚡ The condition is checked before every round — so if it is False from the very start, the body runs zero times. A loop is not a promise that something will happen.

The Infinite Loop ♾️ ⚠️

If nothing in the body ever makes the condition False, the loop never ends. Here i stays 1 forever, so 1 is printed forever:

i = 1 while i <= 5: print(i) # i never changes → runs forever!
🛑 If it happens, stop the program with the red Stop button in PyCharm (or Ctrl + C in a terminal).
✅ Before you run a loop, ask yourself: "what line makes this condition become False?" If you cannot point at it, you have a bug.
4

Collecting a Result — Sum & Maximum

A loop usually builds an answer step by step in a variable that lives outside the loop. Start it before the loop, update it inside, and print it after.

Sum of 5 numbers
total = 0 i = 0 while i < 5: n = int(input('number?')) total += n i += 1 print(total)
Biggest of 5 numbers
biggest = 0 i = 0 while i < 5: n = int(input('number?')) if n > biggest: biggest = n i += 1 print(biggest)
💡 total += n is short for total = total + n. The same shortcut works for -=, *= and /=.
5

Two Reasons to Stop — and in the Condition

A loop condition can be any condition — including and / or. Here the loop runs while there are still students and we have not found a perfect grade yet. It stops on whichever happens first: 100 cannot be beaten, so there is no reason to keep asking.

num_stud = int(input('number of students?')) i = 0 max_grade = 0 while i < num_stud and max_grade < 100: i += 1 grade = int(input('grade?')) if grade < 0 or grade > 100: print('illegal grade') elif grade > max_grade: max_grade = grade print(max_grade)
number of students?3 grade?80 grade?95 grade?60 95
🧠 An if can live inside a loop, and a loop can live inside an if. The indentation tells Python who belongs to whom — every extra level is 4 more spaces.

if vs while — One Word, Huge Difference

Same condition, same indented block. The if version prints once; the while version prints five times:

if — checks once
i = 1 if i <= 5: print(i) i += 1 # prints: 1
while — checks every round
i = 1 while i <= 5: print(i) i += 1 # prints: 1 2 3 4 5

Quick Reference

Every loop needs three things
start the counter before · test it in the condition · change it in the body.
The condition is checked first
If it is False at the start, the body runs zero times.
Infinite loop
Forgot i += 1? The condition never turns False and the program never ends.
Results live outside
Variables like total or max_grade are created before the loop, updated inside, printed after.
