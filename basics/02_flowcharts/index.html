Flowcharts Guide

A flowchart is a visual plan for a program. It shows the order of steps, the data that enters and leaves, and the choices or loops in the logic.

Start

The first step of the flowchart. Every flowchart begins with one START oval.

Stop

The final step. Nothing should happen after STOP.

Input

Use input when the user gives information to the program, like a name or number.

Output

Use output when the program shows something to the user, like an answer or message.

Operation

Use operation for calculations or assignments, like total = price * 3.

If

Use if for a question with different paths, usually YES and NO.

If / Else — Two Paths

The diamond splits the flow into two paths. YES runs the if body; NO runs the else body. Both paths join again afterwards.

Example: if age >= 18 — YES → "You get a Beer", NO → "You get a Coke"

Integer Division (//)

Divides two numbers and drops the decimal part. Use it inside an operation box when you need whole groups.

10 // 3 = 3  ·  7 // 2 = 3  ·  cars = children // 5

Modulus (%)

Returns the remainder after dividing. Most common use: check if a number is even or odd.

8 % 2 = 0 (even)  ·  7 % 2 = 1 (odd)  ·  10 % 3 = 1

While

Use while when steps repeat as long as a condition is true. The flow goes back to the question until the answer becomes NO.

Example: while the number is smaller than 10, add 1 and check again.

Good Flowchart Rules

Use arrows to show the order of the steps.
Use the correct shape for each kind of step.
Keep the text inside each shape short and clear.
